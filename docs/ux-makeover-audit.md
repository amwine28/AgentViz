# AgentViz — UI/UX Makeover Audit

_Multi-agent audit (6 dimension experts → consolidation → 3 directions → synthesis). 47 raw findings → 37 consolidated._

## Recommended direction

**Refined Instrument — elevate the existing "deep-space mission control" language rather than reinventing it: demote the atmosphere FX behind the chrome, push status hues apart so they survive bloom, declutter the 3D labels with screen-space collision + backing plates, add a redundant shape/depth channel for status, and roll out a real token system (color tiers, type scale, tracking ramp, spacing, focus, reduced-motion). Same brand DNA (void radial-gradient, Chakra Petch + IBM Plex Mono, glass panels, corner brackets, phosphor palette) — just disciplined and AA-clean.**

AgentViz is a live, screenshot-worthy multi-agent visualizer; its entire pitch is "watch the swarm think in a deep-space mission-control HUD." The screenshot proves the identity is genuinely good and worth preserving — but it also proves the two failures that undercut the pitch: the six central agents (composer/pattern-miner/signal-parser/synthesis-lead/mission-control/archivist) overprint into an unreadable block (Scene3D.tsx:58-83 always-on labels, fixed (0,11.5,0) offset at :81, depthTest:false at :77, no collision logic), and the whole field is one uniform green-cyan glow with only the lone red scout-bravo legible (--c-run #3fe0ff vs --c-done #6ef7a0 only ~25 hue degrees apart at styles.css:18-20, blown toward white by the white-anchored halo at Scene3D.tsx:50 + UnrealBloomPass at :180-183). Those are exactly the things a "Refined Instrument" direction targets without genericizing the look. "Cinematic Deep Field" (Direction B) adds beautiful nebula layers but spends effort decorating the spectacle when the spectacle's core problem is legibility, not drama — and its darker void/extra fog risk making the already-flat panel elevation worse. "Editorial Clarity" (Direction C) is the safest a11y win but it kills the corner brackets, scanlines, and glow text-shadows — it throws away the brand the screenshot shows is working and would make AgentViz look like every other dark dashboard. Refined Instrument keeps the soul, fixes the two showstoppers, and clears WCAG via tokens. It also reuses patterns the codebase already proves intentional (GraphStats view-gating at App.tsx:134; the .ack-* vocabulary at styles.css:472-475 + ApprovalQueue.tsx:84-94; tabular-nums at styles.css:159,606), so the work extends existing discipline instead of inventing mechanisms.

## Three design directions

### Refined Instrument
Keep the deep-space mission-control soul but rebuild it as a precision instrument: a disciplined token system, a real type/tracking/spacing scale, atmosphere FX demoted behind the chrome, AA-clean contrast, and a 3D field where status reads through shape and depth — not just a uniform phosphor glow.

**Palette:** `#04060d` Deepest background stop / 3D clear color (unchanged — anchors the identity)., `#070b16` Upper radial-gradient stop (unchanged)., `rgba(11,18,34,0.74)` NEW glass fill, slightly lifted from old --glass rgba(8,14,27,0.66) so panels detach over the dark void (addresses flat-elevation finding)., `rgba(7,12,24,0.88)` Heavy glass for top bar + sticky headers (replaces --glass-heavy)., `rgba(125,170,255,0.16)` Hairline HUD rule (nudged up from 0.14 for panel-edge survivability)., `rgba(125,170,255,0.40)` Corner brackets, active borders, focus accents., `#cdd9ef` Primary text / data values (kept bright)., `#8294b4` NEW resting interactive text — buttons, inactive toggles, lane headers (~5.3:1, clears AA). Resolves the dim-until-hover finding., `#6b7d9c` NEW quiet-metadata tier — section labels, stat keys, timestamps, credit heads, run-meta (~3.4:1). Replaces --ink-faint on all TEXT., `#33415c` RETAINED but demoted to DECORATIVE only — hairlines, scrollbar thumb, disabled glyphs, dead-node chips. Never load-bearing text., `#3fe0ff` RUNNING status — the one true cyan. No longer shared by message edges., `#ff9e3d` WAITING — shifted from #ffb454 toward orange to separate it from approval gold under bloom., `#34d17e` COMPLETE — deepened from #6ef7a0 to a true emerald so it survives bloom and reads distinct from running cyan., `#ff5277` ERROR / failure status only — reserved for swarm state + denied acks + error logs., `#e0566f` NEW muted red for interactive-chrome destructive hovers (Stop-all, close-X, Deny) so routine hovers stop competing with a real error node., `#8b9bb4` PAUSED slate (unchanged) — now also gets a legend row., `#ffd166` NEEDS-APPROVAL gold — always a RING form, never a fill, so shape disambiguates it from orange waiting., `#7df3ff` NEW message-channel ice tone — splits the message edge/particle off the running-cyan hue so node-vs-edge channels stop blurring.

**Typography:** Pairing unchanged: Chakra Petch (display voice) + IBM Plex Mono (data/body) — but their JOBS get disciplined. A 4-stop tracking ramp replaces the 10+ literals: --track-display 0.28em (brand, empty-state), --track-label 0.2em (all uppercase micro-labels), --track-ui 0.12em (buttons/chips/toggles), --track-tight 0.04em (mixed-case headlines). Default line-height 1.5 on body cascades to dense blocks (.tc-args, .msg-body, .log-row); collapse the 1.4/1.45/1.5 one-offs onto it. Add font-variant-numeric:tabular-nums to body so every live digit is tabular (no shimmy on tick).

**Spacing:** Adopt a 4px base unit exposed as tokens: --sp-1 4 / --sp-2 8 / --sp-3 12 / --sp-4 16 / --sp-5 24. ## Gaps the audit flagged for follow-up

- Color contrast was asserted from token hex values but not actually measured — none of the audit findings ran a real WCAG contrast calculation against the rendered radial-gradient background (which varies from #070b16 at center to #04060d at edges), and none accounts for the scanline overlay's measured effect; the ~1.9:1 / ~3.9:1 figures are reasonable estimates, not verified numbers.
- Performance was not assessed: Scene3D runs an unconditional requestAnimationFrame loop (Scene3D.tsx:196-213) plus UnrealBloomPass and a 2200-point starfield even when the tab is idle or no agents exist; Graph.tsx and FlowView rebuild/re-layout on every data change. No finding evaluated frame rate, the cost of structureChanged re-layouts, or whether bloom is GPU-heavy on integrated graphics — relevant for a live tool left running during demos.
- No finding examined behavior at large agent counts beyond the 15 in the screenshot — FlowView caps at MAX_ROWS 1500 (FlowView.tsx:15) and 3D has no LOD, but scaling to 50-100 agents (label storm, lane width, force layout, particle emission volume) was not analyzed.
- Loading / transitional / error states for data fetches were not covered: RunPicker (RunPicker.tsx:18-30) shows only a terminal error string with no loading spinner; there is no reconnecting state for the websocket beyond the binary LINK / NO LINK LED, and no stale-data indicator when the relay link drops mid-run.
- Touch / pointer ergonomics untested: targets like the 6px scrollbar (styles.css:47), the 26x22px FLOW filter chips (styles.css:644-646), and the panel-close ✕ are below the ~44px touch-target guideline; no audit considered trackpad/touch use of the orbit controls or 2D drag.
- The 2D Graph has no zoom/pan affordance and the d3 force layout has no drag-to-pin — for larger graphs there is no way to declutter manually; this UX gap parallels the 3D label-collision finding but was only raised for 3D.
- Color of the 2D root node (#7daaff, Graph.tsx:21) and the FLOW spawn color (#7daaff) sit outside the documented status palette and are not in the legend's status rows — a minor palette-coherence gap not flagged.
- Internationalization / long-string robustness: agent names use word-break:break-all (styles.css:324) and fixed truncation, but very long session names, tool names, or non-Latin content were not tested against the fixed 320px panels and 170px lanes.
